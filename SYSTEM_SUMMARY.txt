SYSTEM ARCHITECTURE

Input Data:
├─ ntl_features.csv          (19,280 pincodes × 15 cols)    [VIIRS nighttime lights]
├─ worldpop_features.csv     (19,313 pincodes × 20 cols)    [Population density distribution]
└─ india_pincodes.geojson    (Spatial boundaries)           [Used in merge/analysis]

Processing Pipeline:
│
├─ Step 1: MERGE FEATURES (merge_features.py)
│  Input:  ntl_features.csv + worldpop_features.csv
│  Output: merged_features.csv (19,312 pincodes × 34 cols)
│  Action: Left join on pincode; fill missing NTL with 0
│  Runtime: < 1 second
│
├─ Step 2: AFFLUENCE SCORING (affluence_engg.py)
│  Input:  merged_features.csv
│  Step 2a: Compute 7 cross-signal features (NTL × WorldPop interactions)
│  Step 2b: Compute affluence_raw (weighted sum of signals)
│  Step 2c: Rank against clean pincodes → affluence_percentile [0-100]
│  Step 2d: Assign confidence weights (0.2-1.0)
│  Output: affluence_ranking.csv (19,312 pincodes × 26 cols)
│  Runtime: 2-3 seconds per mode
│  Modes:   default (balanced), econ (economic-heavy), urban (city-development),
│           relative (regional-peer)
│
└─ Step 3: VALIDATION & ANALYSIS (affluence_analysis.py)
   Input:  affluence_ranking.csv
   Output: affluence_analysis_outputs/ (4 interactive HTML plots + console report)
   Runtime: 5-10 seconds
   Checks:
     - Reference pincode validation (known affluent areas)
     - Correlation analysis (7 cross-signals vs affluence)
     - Data quality distribution
     - Contamination summary

WEIGHTING MODES

Mode Selection:
  --mode default     [RECOMMENDED FOR GENERAL USE]
    Balanced weighting across all signals
    Weights: ntl_per_capita=0.25, urban_intensity=0.20, settlement_quality=0.15,
             development_gap=0.15, economic_density=0.10, slum_risk_score=-0.15

  --mode econ        [For economic activity focus]
    Emphasizes NTL intensity and economic footprint
    Weights: ntl_per_capita=0.35, economic_density=0.15
    Use: Industrial hubs, metro growth, economic development zones

  --mode urban       [For urban development focus]
    Emphasizes organized cities, penalizes informal settlements
    Weights: urban_intensity=0.30, settlement_quality=0.20, slum_risk_score=-0.20
    Use: Smart city planning, residential quality, anti-slum metrics

  --mode relative    [For regional peer comparison]
    Emphasizes relative standing within regions/states
    Weights: development_gap=0.30 (uses state percentiles)
    Use: Within-state inequality, regional competition, relative prosperity

CONTAMINATION MANAGEMENT

When to exclude/downweight:

  industrial_flag == 1 (1,553 pincodes, 8%):
  └─ High NTL from gas flares/oil refineries (not residential activity)
     Confidence: ×0.3
     Use case: If analyzing residential affluence, exclude
               If analyzing economic activity, use as-is

  sparse_sample_flag == 1 (7,995 pincodes, 41.4%):
  └─ <10 pixels in population sample (distribution features unreliable)
     Confidence: ×0.6
     Use case: For these pincodes, prefer raw features (log_pop_density)
               over distribution features (pop_cv, pop_gini)

  uninhabited_flag == 1 (4 pincodes, 0.02%):
  └─ No population detected (water bodies, deserts, protected areas)
     Confidence: ×0.2
     Use case: Generally exclude from human-focused analysis

  is_contaminated == 1 (9,526 pincodes, 49.3%):
  └─ Any of the above flags are set
     Recommendation: Can use but apply appropriate confidence-weighting
                    For strict "verified affluent" list, filter to 
                    confidence >= 0.9 AND affluence_percentile >= 90

Common Questions:
  Q: Why is my known affluent area not in top 1%?
  A: Check confidence score. Sparse sample areas (confidence=0.6) may have
     unreliable distribution features but raw NTL/density are still valid.

  Q: Should I use weighted or unweighted percentile?
  A: Use affluence_percentile for ranking. Use 
     affluence_percentile_weighted for filtering/targeting (stricter).

  Q: How do I choose between modes?
  A: Default is best for general use. Use econ for economic focus, urban
     for city development, relative for within-state comparison.

  Q: What confidence level is safe to use?
  A: confidence >= 0.9 is high-confidence. confidence >= 0.6 is usable
     with caution. confidence < 0.6 requires validation.
